## 与原版相比的主要优化

### v1.2.2 更新摘要

- 修复超级抽奖选车兜底：当“设计与涂装 / 设计与喷漆”入口失败时，会先可靠退回车辆菜单并重新进入“我的车辆”，再执行普通选车流程，避免兜底触发后误进其他入口。
- 保留首次失败立即普通选车兜底、累计两次失败后本轮后续强制普通选车的策略，降低循环卡死概率。
- 同步版本号至 1.2.2。

### v1.2.1 更新摘要

- 修复移除车辆模块在“无可删车辆”场景下可能停留在删车步骤循环不前的问题：当连续扫描后确认没有可删除候选时，会自动跳过当前删车步骤并进入下一轮大循环，避免流程卡死。
- 延续并强化删车保护策略：检测到全新标签时直接跳过，不执行删除；若出现“可删模板”和“全新标签”冲突，按全新车辆处理并跳过。
- 优化删车过程日志与诊断字段，便于复盘“为什么跳过/为什么未删”。

### v1.2.0 更新摘要

- 新增 **【测试】模板识别阈值调试窗口**：可查看每个模板的用途、分类、组合关系、最近识别分数和当前阈值，并支持按组合同步调整阈值。该功能仍处于测试阶段，尚未完成完整场景验证；如果运行中遇到模板识别问题，请结合日志和错误诊断文件谨慎调整。
- 优化移除车辆逻辑：第一次定位到筛选结果末页后，成功删除车辆的后续循环不再重复 PageDown；删除前会先做全新标签强保护，再做非全新正向确认。只要检测到 `FreshTagText.png` 或卖车专用全新模板命中，就会直接跳过当前车辆；即便普通可删模板同时命中，也会按全新车辆处理，避免误删用户刚购买但尚未处理的车辆。删车保护与 CJ 选车的全新模板阈值也已拆分，调试窗口单独调整删车阈值时不会再影响 CJ。
- 新增刷技术点/超级抽奖买车前的 **CR 点不足兜底**：默认开启。买 22B 前会按剩余买车数量预留 CR；不足时自动启动刷 CR 点流程，刷到足够完成本轮买车循环后继续刷技术点/超级抽奖。

### 1. 循环跑图逻辑优化

- 支持 S2 900 与 S1 790 两种刷图车辆配置。
- S1 790 模式增加车辆共享代码与适配地图分享代码提示。
- 增强刷图车辆模板候选与识别流程，降低选错车概率。
- 新增赛事内卡住检测：超过设定时间未完成时，先松开 `W`，按住 `S` 倒车脱困，再继续前进。
- 新增单场赛事超时重开逻辑，倒车兜底达到上限后进入重开/恢复流程。
- 增强断点恢复：识图失败、菜单异常、游戏窗口丢失或流程中断时，会尽量回到可继续执行的状态。

### 2. 批量买车逻辑优化

- 强化车辆收藏、品牌定位、目标车辆选择和购买确认流程。
- 买车数量可同步到后续抽奖/移除车辆数量，减少流水线配置错误。
- 模板图和等待参数做了更细拆分，适配不同加载速度和 UI 状态。
- 与超级抽奖模块联动：当抽奖阶段找不到目标车辆时，可跳回买车模块补车。

### 3. 超级抽奖选车逻辑优化

- 选车前先应用 **重复项 + B级 + 全轮驱动 + 传奇** 筛选，缩小目标范围。
- 增加全新标签识别，优先寻找带全新标记的 22B。
- 通过 PageUp / PageDown 回到列表前端或初始页面，多轮扫描目标车辆。
- 默认尝试从“设计与涂装 / 设计与喷漆”入口快速选车；若连续失败，会回退到普通选车流程。
- 为保证快速选车正常，建议提前手动进入“设计与涂装 / 设计与喷漆”，并将提示弹窗选择“不再显示此消息”。
- 找不到目标车时不会盲目继续点技能，而是转回买车模块补充车辆。
- 技能矩阵仍支持自定义方向路径，适配不同车辆熟练度树。

### 4. 新增移除车辆模块

- 新增“移除车辆”模块，可批量移除筛选结果中的车辆。
- 默认筛选条件为 **重复项 + B级 + 全轮驱动 + 传奇**，大概率对应 22B。
- 删除前有强提醒，要求用户人工审核筛选结果。
- 删除逻辑会尝试确认车辆操作菜单、点击“从车库移除车辆”，并等待界面恢复。

> 移除车辆模块存在误删风险。启动前必须人工确认筛选结果；如果有其他车辆混入，请先改装至非 B 级或不要启动删车。

### 5. 大循环流水线优化

优化版支持四个模块串联：

**跑图 -> 买车 -> 超级抽奖 -> 移除车辆 -> 下一轮**

可配置：

- 每个模块执行次数
- 每个模块完成后是否继续
- 下一步跳转到哪个模块
- 大循环总轮数
- 三/四模块完成后是否重新计数进入下一轮

当某个模块发生可恢复错误时，程序会尝试恢复后重新进入当前模块，避免整条流水线直接报废。

### 6. 新增刷 CR 点模块

- 新增在线劲敌刷 CR 点流程。
- 支持五菱、丰田两类刷 CR 车辆。
- 支持配置不同车辆的单圈估算秒数。
- 支持周期结算，按设定圈数退出赛事结算后再进入下一轮。
- 处理服务器错误、无法更新劲敌数据等常见阻塞状态。
- 增加刷 CR 守护：检测手柄失联提示、赛事设置界面、未知状态，并尝试恢复。
- 增加 CR 区域截图和诊断记录，方便后续排查识别问题。

### 7. 稳定性与诊断优化

- 新增进程守护，可按设定间隔检测 `forzahorizon6.exe` 是否仍在运行。
- 支持游戏闪退后的自动重启测试功能，可用自定义命令拉起游戏。
- 关键步骤可开启操作验证重试：执行操作后等待模板确认，失败则重试。
- 失败时自动保存现场截图、诊断 JSON 和日志上下文。
- 默认诊断目录为 `diagnostics/年-月-日/`。
- 停止脚本时会主动释放方向键、Enter、Esc、Backspace、Space、持续按住的 `W` 等按键。
- 支持热键 **F8** 紧急停止。

## 项目简介

FH6Auto 是一个围绕游戏界面自动识别与流程控制设计的桌面自动化工具。  
项目以 **图像识别** 作为流程引导核心，尽量避免纯按键脚本“盲操作”带来的失控风险。

工具通过以下方式实现自动执行：

- 截图识别当前游戏状态
- 动态判断是否进入目标页面
- 触发键盘 / 鼠标操作
- 在出错时自动尝试恢复运行环境

相比传统纯按键脚本，本项目在稳定性、适应性和可控性方面更好。

## 功能模块

### 1. 循环跑图

- 自动进入菜单
- 自动切换创意中心 / EventLab
- 自动输入蓝图分享代码
- 自动匹配目标车辆
- 自动循环开始赛事
- 支持 S2 900 / S1 790 车辆配置
- 支持跑图卡住时自动倒车兜底
- 支持赛事超时重开与断点恢复
- 支持按设定次数重复执行

### 2. 批量买车

- 自动进入车辆收藏
- 自动定位目标品牌
- 自动选择指定车辆
- 自动重复购买
- 支持按设定次数批量执行
- 可与抽奖、移除车辆数量联动

### 3. 超级抽奖

- 自动进入买车流程
- 自动定位目标车辆
- 自动进入升级 / 熟练度界面
- 自动按技能矩阵点技能
- 支持筛选重复项+B级+全轮驱动+传奇车辆
- 支持设计与涂装入口失败后回退普通选车
- 支持技能点耗尽后自动结束模块

### 4. 移除车辆

- 自动进入车辆菜单
- 自动应用目标筛选条件
- 自动定位筛选结果尾部车辆
- 自动打开车辆操作菜单
- 自动移除指定数量车辆

### 5. 刷 CR 点

- 自动进入在线劲敌赛事
- 自动选择刷 CR 车辆
- 自动开始赛事并开启自动驾驶
- 支持周期结算与循环执行
- 支持服务器错误和未知状态恢复

### 6. 大循环挂机

支持模块串联形成完整流水线：

**跑图 -> 买车 -> 超级抽奖 -> 移除车辆 -> 重置计数 -> 下一轮**

可配置：

- 是否继续到下一模块
- 下一模块编号
- 是否在模块完成后重新循环
- 总循环次数
- 近似无限挂机执行

## 使用方法

### 1. 启动前准备

#### 车辆准备

请先购买用于跑图和流程消耗的车辆，并完成以下准备：

- 刷图车：**斯巴鲁 Impreza 22B-STi Version**
- S2 900 模式：调校至 **S2 900**，加入收藏，不要有任何涂装
- S1 790 模式：使用程序提示的车辆共享代码和适配地图分享代码，加入收藏，不要有任何涂装
- 建议车库里除了 22B 以外的斯巴鲁车辆换成白色等明显不同涂装，提高模板识别稳定性
- 建议提前点赞跑图车辆调校和跑图蓝图
- 建议提前点满用于刷技术点的 22B 技能树

#### 游戏设置建议

- 游戏已经正常启动
- 游戏语言设置为 **简体中文**
- 键盘输入法切换为 **英文键盘**
- 跑图/抽奖流水线建议设置为 **所向披靡 / 自动转向 / 自动挡**
- 脚本会通过 **设计与涂装 / 设计与喷漆** 页面快速选车；为保证选车正常，请提前手动进入该界面，并将进入时触发的提示弹窗选择 **不再显示此消息**
- 刷 CR 点若追求收益，可按程序提醒切换到对应驾驶辅助设置
- 游戏界面尽量保持稳定
- 不要随意切换到其他窗口，以免影响识图结果

- 更方便维护与迭代

## 防错与恢复说明

如果执行过程中发生以下情况：

- 图像识别失败
- 当前界面异常
- 流程中断
- 游戏闪退
- 服务器错误
- 目标车辆未找到

程序会优先尝试：

1. 检查游戏进程
2. 聚焦游戏窗口
3. 退回漫游 / 菜单
4. 保存失败现场
5. 自动恢复到可继续执行的状态

循环跑图模块包含比赛内本地兜底：

- 如果当前圈连续超过设定秒数仍未检测到完成界面
- 脚本会先松开 `W`
- 按住 `S` 指定秒数让车辆倒车脱困
- 再恢复持续按住 `W`
- 单圈最多执行 2 次本地兜底，之后升级到重开/断点恢复流程

如果启用了自动重启：

- 会尝试使用预设命令重新启动游戏
- 运行中会按“进程检测秒数”主动检测 `forzahorizon6.exe` 是否消失
- 只有在开启自动重启时，进程守护线程才会触发重启恢复
- 并在识别到继续界面后恢复流程

> 自动恢复不能保证 100% 成功，但相比无保护脚本，稳定性会更高。

当脚本遇到当前逻辑无法处理的情况时，会自动保存一份失败现场：

- 当前游戏画面截图
- 同名诊断 `.json` 文件
- `bot_log.txt` 中对应的事件记录

默认保存目录为：

- `diagnostics/年-月-日/`

如果需要开发者排查问题，建议把以下内容一起提供：

- `bot_log.txt`
- 对应日期下的 `diagnostics` 目录

## 风险提醒

- 本工具依赖图像识别，任何分辨率、缩放、画质、UI 状态、语言设置、加载速度差异都可能导致识别失败。
- 自动买车、自动抽奖和移除车辆都涉及账号资产变化，请先少量测试。
- 移除车辆模块尤其需要人工审核筛选结果，确认没有其他珍贵车辆混入。
- 自动重启、刷 CR 守护、操作验证重试属于提高稳定性的辅助机制，不能保证每次都恢复成功。
- 本优化版可能滞后于原作者更新，请优先按本 README 和本仓库 Release 说明使用。

## 项目亮点

### 基于 Python 图像识别

本项目并不是简单的固定坐标点击器，而是结合以下技术进行界面识别：

- `pyautogui` 截图
- `opencv-python (cv2)` 模板匹配
- `numpy` 图像处理

对当前游戏界面进行识别后，再决定下一步动作，因此相比纯脚本具有更高的适应性与可控性。

### 尽量减少“盲操”风险

传统脚本常见问题包括：

- 不判断页面状态
- 一直机械按键
- 一旦卡界面就彻底跑偏

本项目尽量通过识图确认：

- 当前是否在漫游界面
- 是否进入主菜单
- 是否找到指定按钮
- 是否已进入升级页
- 是否出现继续游戏 / 欢迎界面 / 重启页面
- 是否处于刷 CR 已知赛事状态

只有识别到正确状态，才执行后续动作。

### 多层防错误机制

项目内包含多层保护逻辑，例如：

#### 1. 状态验证

关键步骤前会确认：

- 是否成功进入菜单
- 是否识别到目标页面
- 是否找到指定图片模板
- 是否仍在运行状态

#### 2. 中断恢复

当模块执行失败时，会尝试：

- 检测游戏进程是否仍存在
- 自动聚焦游戏窗口
- 自动尝试退回菜单 / 漫游
- 从中断模块继续执行剩余流程

#### 3. 自动重启机制（测试功能）

当检测到游戏进程不存在时，可选择：

- 执行自定义启动命令
- 等待游戏重新启动
- 自动识别欢迎页 / 继续游戏界面
- 尝试重新回到游戏并恢复执行

#### 4. 按键强制释放

脚本停止时会主动释放：

- 方向键
- Enter / Esc / Backspace / Space
- 持续按住的 `W` 键等

避免因为异常退出导致“卡键”。

### 支持无限循环挂机

你可以把它理解成一个可编排的小型挂机流水线：

- 先跑图刷收益
- 再自动买车
- 然后自动点熟练度拿抽奖
- 再按筛选条件移除车辆
- 完成后进入下一轮

配合总循环数设置，可以长时间无人值守运行。  
如果将循环次数设置得足够高，也可视作“无限循环挂机”。

### 技能矩阵可自定义

超级抽奖模块支持手动设置技能点路径。  
用户可以通过 UI 中的方向按钮，自定义：

- 上
- 下
- 左
- 右

形成一条技能路径矩阵，用于适配不同车辆的熟练度树。

## 技术实现

本项目主要基于以下 Python 技术栈：

- **customtkinter**：现代化桌面 UI
- **opencv-python**：模板匹配 / 图像识别
- **numpy**：图像数组处理
- **pyautogui**：截图与基础自动化
- **pydirectinput**：更适合游戏场景的输入模拟
- **ctypes / win32gui**：窗口聚焦、DPI 适配、底层输入
- **threading**：后台任务执行
- **PIL**：资源图像加载
- **requests**：版本更新检查
